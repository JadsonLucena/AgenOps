# Topology Layer

This layer makes team ownership and interaction explicit.

## Files
- `team-types.md`
- `ownership-map.yaml`
- `interaction-modes.md`
- `flow-model.md`

## Why it exists
A technically correct kit can still fail if the organizational topology is ambiguous.

This layer ensures the system is:
- operationally viable
- easy to adopt
- low-coupling by design
- aligned with flow efficiency
