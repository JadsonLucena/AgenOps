# Versioning Policy

## Rules

### 1. Every release must be versioned
No unversioned deployments.

### 2. Version must match commit history
Derived from conventional commits.

### 3. Tagging
- use git tags for releases
- format: vMAJOR.MINOR.PATCH

### 4. Changelog generation
Automatically generated from commits.

### 5. Breaking changes require:
- MAJOR bump
- explicit decision record
- migration notes

### 6. Traceability
Every version must link to:
- decisions
- artifacts
- evidence bundle
