# Experience Module

## Purpose

Make the kit easy to enter, navigate, and apply in daily use.

## What it provides

- phase maps
- quick flows
- next-step guidance
- command maps
- menu patterns
- onboarding guidance
- state routing
- persistent harness state
- formal sub-agent routing
- brainstorm routing
- spec document design routing
- design chunk routing
- context budget awareness
- pruning and snapshot guidance

## Design rule

The experience layer must reduce cognitive load without hiding the source of truth.
It should guide action, not replace governance.
