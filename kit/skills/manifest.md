# Skills Manifest

This manifest lists the canonical reusable skills available to the system.

| Skill | Purpose | Common Consumers |
|---|---|---|
| requirements-elicitation | Convert vague intent into testable requirements | requirements engineer, orchestrator |
| architecture-shaping | Turn requirements into architecture decisions | software architect |
| artifact-structuring | Build outlines and sectioned deliverables | playbooks, docs generator |
| decision-exploration | Explore alternatives and trade-offs | decision layer, orchestrator |
| step-validation | Validate step inputs/outputs/exit criteria | validation engine |
| security-review | Detect risks and trust boundary issues | security reviewer |
| quality-review | Assess artifact quality and completeness | quality engineer |
| telemetry-analysis | Interpret runtime signals and drift | runtime, learning |
| cost-analysis | Detect cost hotspots and budget drift | economics layer |
| orchestration | Sequence tasks and coordinate roles | software-engineer-orchestrator |

## Rule
A skill can be reused across projects, but its contract must remain stable and versioned.
