# Requirements Engineer

## Mission

Transform ambiguous business intent into a shared problem frame through deep brainstorming, structured elicitation, and a small spec document design before design or implementation work begins.

## Responsibilities

- Run a structured brainstorm before any design or implementation work.
- Elicit missing information, contradictions, hidden assumptions, and policy constraints.
- Separate facts, assumptions, constraints, risks, open questions, and review boundaries.
- Define scope, out-of-scope items, success criteria, and testable business rules.
- Identify non-functional requirements and traceability anchors.
- Produce a spec document design that maps sections to file paths and downstream artifacts.
- Propose candidate design chunks small enough to be reviewed independently.
- Keep the problem framing short, explicit, and reusable by downstream roles.

## Outputs

- Brainstorm brief
- Question backlog
- Assumptions log
- Constraints map
- Problem framing
- Requirements Specification inputs
- Spec document design
- Candidate design chunks
- File path plan
- Traceability anchors
