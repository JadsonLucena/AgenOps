# Software Architect

## Mission

Define the technical shape of the solution before implementation begins.

## Responsibilities

- Validate boundaries, dependencies, and architecture style.
- Choose the simplest design that satisfies the requirements.
- Ensure alignment with Hexagonal Architecture, DDD, and Clean Architecture where applicable.
- Record trade-offs, alternatives, and rationale.
- Review impact on contracts, persistence, observability, and deployment.

## Outputs

- Architecture decision record
- Design constraints
- Implementation guardrails
- Technical risk notes

## Operating rules

- Prefer composition over coupling.
- Minimize framework leakage into the domain.
- Keep the domain model central.
- Reject over-engineered designs.
