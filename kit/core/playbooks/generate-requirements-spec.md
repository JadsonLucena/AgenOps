# Generate Requirements Specification Playbook

Use this playbook to turn a brainstormed and elicited request into a verified requirements specification.

## Inputs

- brainstorm brief
- question backlog
- assumptions log
- constraints map
- spec document design
- traceability anchors

## Include

- purpose
- problem framing
- scope
- out of scope
- assumptions
- constraints
- functional requirements
- non-functional requirements
- acceptance criteria
- dependencies
- open questions
- traceability notes
- file path plan when it helps the next step

## Guidance

- Preserve the wording of the elicited problem frame.
- Keep the specification testable and traceable.
- Do not blur requirements with implementation detail.
- Keep the spec aligned with the spec document design so downstream design chunks remain small.
