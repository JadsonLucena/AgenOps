# Sandbox Policy

## Scope

Keep default execution constrained and expand it only when needed.

## Requirements

- Keep default execution constrained.
- Escalate only when needed.
- Prefer reproducible, deterministic steps.
- Keep the boundary between data and instructions clear.

## Exceptions

- Deviations require explicit rationale and approval when they affect shared behavior, security, or compatibility.

## Evidence

- review notes
- validation output
- linked change record
