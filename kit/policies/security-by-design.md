# Security by Design

## Scope

Embed security controls into the design rather than adding them after implementation.

## Requirements

- minimize trust
- validate inputs
- isolate boundaries
- protect secrets
- least privilege
- fail safe
- log meaningfully without leaking sensitive data
- review security impact early, not after implementation

## Exceptions

- Deviations require explicit rationale and approval when they affect shared behavior, security, or compatibility.

## Evidence

- review notes
- validation output
- linked change record
