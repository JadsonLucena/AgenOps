# Token Budget Policy

Set a budget by task type and phase.

## Guidance

- use more context for architecture, requirements, and spec design
- use less context for focused fixes and reviews
- prefer concise outputs
- avoid redundant re-explanations
- stop adding context once enough evidence is available
- when the budget is tight, compact before introducing new material
- prefer summaries and snapshots over raw accumulation

## Rule

Token budget and context budget should be coordinated.
If one is approaching the limit, the harness should compact before expanding the active context.
