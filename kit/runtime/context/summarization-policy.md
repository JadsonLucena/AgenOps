# Summarization Policy

Summaries should preserve decision value and rehydration value.

## Include

- task goal
- phase
- what changed
- why it changed
- what remains open
- what is risky
- what was verified
- current snapshot pointer
- next action

## Avoid

- repeated background text
- unrelated history
- full log dumps
- raw reasoning traces
- duplicate excerpts of durable docs

## Format

Prefer:
- short bullets
- stable artifact references
- version numbers
- explicit blockers and open questions
