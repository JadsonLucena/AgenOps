# Definition of Done

## Purpose

Define the minimum completion criteria for a change.

## Items

- [ ] Requirements are clear or open questions are explicitly tracked.
- [ ] Architecture direction is approved or recorded.
- [ ] Tests exist for the intended behavior.
- [ ] Code follows the intended architecture and design rules.
- [ ] QA, Security, Code Review, and DevOps findings are addressed or justified.
- [ ] Only triggered documentation was updated.
- [ ] Decision summaries and traceability artifacts are complete.
- [ ] The change is ready for merge or release.

## Pass condition

All items are checked or have documented exceptions.
