# Secure by Design Principles

## Operational posture
Security should be built into the system design, not bolted on afterward.

## Principles
- make secure behavior the default
- minimize exposed attack surface
- reduce blast radius
- make risky actions observable
- keep security decisions explicit
- provide safe defaults for agents and users
- ensure transparency and accountability for exceptions

## Application to this kit
- policies are first-class
- trust boundaries are explicit
- permissions are schema-driven
- agent autonomy is bounded
- approvals are evidence-based
