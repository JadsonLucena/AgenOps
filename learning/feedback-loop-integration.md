# Feedback Loop Integration

## Inputs
- runtime telemetry
- validation failures
- security findings
- review reports
- incidents

## Outputs
- updated contracts
- new validation rules
- refined policies
- improved templates
- adjusted agent behavior

## Integration points

- validation → emits signals to learning
- telemetry → feeds metrics
- orchestrator → triggers learning review when thresholds exceeded

## Trigger examples

- drift > threshold
- repeated validation failure
- recurring review comments
- incident severity high

## Governance rule

No major pattern change without:
- evidence from learning layer
- documented decision
- versioned update
